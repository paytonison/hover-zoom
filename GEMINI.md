# GEMINI.md

## Introduction
This document outlines the purpose, capabilities, and usage of the Gemini CLI agent within this project.

## Purpose
The Gemini CLI agent is designed to assist with various software engineering tasks, including bug fixes, feature development, refactoring, and code explanation. It operates by understanding natural language requests, strategizing, planning, implementing changes, and verifying results using available tools.

## Capabilities
- **Code Understanding:** Analyzes project structure, dependencies, and conventions.
- **Task Execution:** Performs file operations, runs shell commands, and modifies code.
- **Testing & Verification:** Identifies and runs tests, performs linting and type-checking.
- **Interactive Assistance:** Communicates progress, seeks clarification, and confirms actions.

## Usage
To interact with the Gemini CLI agent, provide clear and concise instructions for the desired task. The agent will then propose a plan, execute it, and report on the outcome.

### Example Commands:
- "Fix the bug in `src/main.js` that causes a null pointer exception."
- "Implement a new feature to allow user authentication."
- "Refactor the `data_processing` module for better performance."
- "Explain the `calculate_total` function."

Any specific configurations or environment variables related to the Gemini agent will be detailed here. (Currently, none).

## Development Guidelines
When working with the Gemini agent, ensure that all instructions are precise and provide sufficient context. The agent strives to adhere to existing project conventions and best practices.
